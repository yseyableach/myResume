我在 AIDC / Chthonia 裡的價值，不只是完成單一功能，而是理解整個 AI Data Center 平台從產品介面、資源治理、GitOps 交付、Kubernetes runtime 到可靠性驗收的 whole picture，並負責把其中的核心平台能力落地成可部署、可驗證、可維運的系統。

平台定位

AIDC / Chthonia 不是單純的 Kubernetes 部署專案，而是一個將 GPU / IDC / Container / Quota / User / Project 等資源產品化的 AI Data Center 平台。平台前台包含兩個主要使用介面：Admin Console 與 Service Portal。

Admin Console 提供平台管理員管理組織、部門、專案、成員、IDC、Cluster、Quota、Billing 與系統設定；Service Portal 則提供 AI Researcher / MLOps 使用者查看 Project、Quota、Container、資源狀態與服務入口。

我負責創造的價值

產品與資源治理

開發 IDC、Organization、Container 等資源管理模組，支援資源 CRUD、相依性檢查、Portal 整合與 cluster 連線驗證。

Kubernetes Runtime 與部署

負責 Kubernetes 基礎設施與多環境部署，支援 Proxmox、AWS 與本機 MicroK8s，讓團隊可快速部署、驗證與展示平台功能。

GitOps 交付流程

建立 portal / portal-helm 分離的部署模式，CI 產出版本化 image，CD 透過 Argo CD 手動選版與同步，避免 GPU 測試環境被不必要部署佔用。

核心服務與可靠性

協助整合 IAM / SSO、API Gateway、儲存、資料庫與監控能力，並納入 HA、資料完整性與 GPU 通訊效能驗證。

交付流程設計

由於 GPU 環境昂貴且共享，平台交付流程不能盲目採用全自動部署。我參與設計以 portal repo 負責 application code 與 image 產出，portal-helm repo 負責 Helm chart 與部署設定的分離模式。CI 負責產出版本化 image，CD 則透過 Argo CD 手動選版與同步，讓團隊可以快速切換測試版本，同時避免每次 merge 都覆蓋或佔用 GPU 測試環境。

可靠性與驗收

AIDC 的價值不只在於服務能跑起來，而是要能被驗收。平台驗收涵蓋功能測試、端對端 UAT、Cluster HA、VM 故障恢復、資料完整性、GPU 算力狀態接續，以及 NCCL / RDMA GPU 通訊效能測試。這代表平台不只是一般 Web Portal，而是具備 AI 訓練工作負載支援能力與基礎設施可靠性驗證的內部 AI Data Center 平台。
