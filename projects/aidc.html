平台定位

AIDC / Chthonia 是一套面向 AI Data Center 的內部平台，目標是將 GPU、IDC、Cluster、Container、Quota、User 與 Project 等資源整理成可管理、可部署、可觀測的服務。

平台包含兩個主要介面：Admin Console 與 Service Portal。Admin Console 供平台管理員管理組織、部門、專案、成員、IDC、Cluster、Quota、Billing 與系統設定；Service Portal 供 AI Researcher / MLOps 使用者查看所屬 Project、Quota、Container、資源狀態與服務入口。

參與範圍

資源管理模組

參與 IDC、Cluster、Container 等資源管理功能，包含 CRUD、相依性檢查、Portal 整合與 cluster 連線驗證。

Kubernetes 與多環境部署

建置與維護 Kubernetes 相關部署環境，支援 Proxmox、AWS 與本機 MicroK8s，協助團隊進行功能驗證與展示。

GitOps 交付流程

參與 portal / portal-helm 分離的部署流程。CI 產出版本化 image，CD 透過 Argo CD 進行手動選版與同步。

核心服務整合

協助整合 IAM / SSO、API Gateway、儲存、資料庫與監控相關能力，支援平台權限控管、服務入口與維運觀測。

交付流程

平台採用 application repo 與 deployment repo 分離的方式：portal repo 負責應用程式程式碼與 image 產出；portal-helm repo 負責 Helm chart 與部署設定。當 feat/* 合併至 dev 後，CI pipeline 會建立版本化 image 並推送至 GHCR。

由於 GPU 測試環境資源有限且屬共享環境，實際部署透過 Argo CD 手動選版與同步。此方式可保留部署控制點，並讓團隊在需要時切換不同測試版本。

可靠性與驗證

平台驗收不只包含功能測試，也涵蓋端對端 UAT、Cluster HA、VM 故障恢復、資料完整性、GPU 算力狀態接續，以及 NCCL / RDMA GPU 通訊效能測試。這些測試用於確認平台在功能、部署、恢復與 AI 工作負載支援上的基本可用性。
